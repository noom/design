UI Interface Guidelines

These guidelines are intended to help standardize mockups and design for Noom's core application: Noom Coach. This does not include each individual element or interaction in the app, but rather aims to provide the design patterns which make up the core Noom Coach Features.

General

The Noom design system is comprised of a series of page design elements and modules which work together to shape the Noom experience. Use the rules outlined in this design system to create new features or functionality to ensure a consistent user experience. The design team uses specific language when referring to specific locations/modules/screens in the app. The figure below describes these nicknames.

Color

While Noom has a variety of colors in its palette, the colors used in the app are limited to the primary color palette. The exception to this rule are the in-app illustrations, which can included colors from the extended palette. The main app color is pbr.

Primary Color

The primary color used throughout the app is pbr. Note that the primary in-app color does not include the various shades of pbr; only pbr itself.

Primary Color Palette

The primary color palette consists of 10 colors, each with 4 shades. Keep in mind readability and mood-associated colors when selecting which shade/color to use - i.e. using cherry or orange may be associated with a warning.
